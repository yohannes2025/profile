# 📚 About the Project

This project is a **professional portfolio platform** designed to showcase software engineering skills, technical projects, professional experience, and achievements.

Rather than being a simple static website, this application is built as a complete **enterprise-grade full-stack solution**.

The project demonstrates modern software engineering practices including:

- Component-based frontend architecture
- RESTful API development
- JWT Authentication
- Responsive UI/UX
- Internationalization (i18n)
- Dark/Light themes
- Docker containerization
- CI/CD automation
- Cloud deployment
- Performance optimization
- Secure backend architecture

The portfolio is intended to serve both as a personal website and as a demonstration of production-ready software engineering skills for recruiters, employers, and clients.

# 🔒 Backend Environment Variables (.env)

Create a `.env` file inside the `backend/` directory.

```env
####################################################
# Django
####################################################

SECRET_KEY=your-secret-key
DEBUG=True

ALLOWED_HOSTS=localhost,127.0.0.1

DJANGO_SETTINGS_MODULE=config.settings

####################################################
# Database
####################################################

DB_ENGINE=django.db.backends.postgresql

DB_NAME=portfolio_db
DB_USER=postgres
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432

####################################################
# Email
####################################################

EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend

EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True

EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password

DEFAULT_FROM_EMAIL=your-email@gmail.com
CONTACT_EMAIL=your-email@gmail.com

####################################################
# JWT Authentication
####################################################

JWT_ACCESS_TOKEN_LIFETIME=60
JWT_REFRESH_TOKEN_LIFETIME=7

####################################################
# Redis
####################################################

REDIS_URL=redis://localhost:6379/0

####################################################
# Celery
####################################################

CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0

####################################################
# Cloudinary
####################################################

CLOUDINARY_CLOUD_NAME=your-cloud-name

####################################################
# Security
####################################################

####################################################
```

# 🔐 Security Best Practices

Never expose:

- SECRET_KEY
- Database password
- SMTP password
- Cloudinary secret
- JWT secret
- API tokens
- Access credentials

Always:

- Use environment variables
- Enable HTTPS
- Rotate secrets periodically
- Restrict allowed hosts
- Restrict CORS origins
- Keep dependencies updated

# 📧 Gmail SMTP Setup

To send emails using Gmail:

1. Enable **2-Step Verification** on your Google account.
2. Generate an **App Password**.
3. Use your Gmail address for `EMAIL_HOST_USER`.
4. Use the generated App Password for `EMAIL_HOST_PASSWORD`.

Do **not** use your normal Google account password.

# 🔑 Authentication Endpoints

## Register User

```http
POST /api/users/register/
```

Request

```json
{
  "username": "johnsmith",
  "email": "john@example.com",
  "password": "StrongPassword123!"
}
```

Successful Response

```json
{
  "id": 1,
  "username": "johnsmith",
  "email": "john@example.com"
}
```

---

## Login

```http
POST /api/users/login/
```

Request

```json
{
  "email": "john@example.com",
  "password": "StrongPassword123!"
}
```

Response

```json
{
  "access": "JWT_ACCESS_TOKEN",
  "refresh": "JWT_REFRESH_TOKEN"
}
```

Request


Response
